## Problem Statement

The Telegram bot's update handler spawns unlimited goroutines without any concurrency control. Under high load (message spam, DoS attempt), this can exhaust system resources.

**Location**: `internal/adapter/telegram/bot.go:208`

**Current Code**:
```go
for _, update := range updates {
    b.offset = update.UpdateID + 1
    go b.handleUpdate(ctx, update)  // ← PROBLEM: unlimited goroutines
}
```
